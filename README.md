# Alexa Ping Me

A minimal Alexa skill that sends proactive notifications to Echo devices, making the yellow/orange ring light turn on.

## What It Does

Uses the [Alexa Proactive Events API](https://developer.amazon.com/en-US/docs/alexa/smapi/proactive-events-api-reference.html) with the `AMAZON.MessageAlert.Activated` schema to push notifications without the user making a voice request first. The notification shows "You have N new message(s) from [sender]" on the device and triggers the ring light.

## Prerequisites

- An [Amazon Developer account](https://developer.amazon.com/alexa) with a skill configured for proactive events
- LWA (Login with Amazon) credentials — Client ID and Client Secret from your skill's Permissions page
- [ask-cli](https://www.npmjs.com/package/ask-cli) (`npm install -g ask-cli`)
- Node.js (for the notification script)
- Python 3 (for the skill server — no pip dependencies)
- A publicly accessible HTTPS endpoint for the skill server

## Setup

### 1. Configure environment

```bash
cp .env.example .env
# Edit .env with your LWA credentials and user ID
```

### 2. Deploy the skill

If you already have a skill ID, update the manifest with your endpoint:

```bash
ask smapi update-skill-manifest \
  -s <YOUR_SKILL_ID> -g development \
  --manifest "$(cat manifest-put.json)"
```

### 3. Start the skill server

```bash
python3 server/alexa_skill_server.py 3000
```

This must be reachable at a public HTTPS URL. Expose it however you like (reverse proxy, tunnel, etc.).

### 4. Enable the skill

- In the Alexa app, find your development skill and enable it
- Go to Settings → Manage Permissions → toggle notifications ON

### 5. Send a notification

```bash
source .env

# To all subscribed devices
./scripts/ping-me-multicast.sh "Hello" 1

# To a specific device (requires ALEXA_USER_ID)
./scripts/ping-me.sh "Hello" 1
```

## Project Structure

```
server/alexa_skill_server.py   Zero-dependency Python skill endpoint
scripts/send-notification.js    Sends proactive events via the Alexa API
scripts/ping-me.sh              Unicast convenience script (one device)
scripts/ping-me-multicast.sh    Multicast convenience script (all devices)
manifest-put.json               Skill manifest template (SMAPI format)
skill-package/models/           Interaction models (en-US, it-IT)
lambda/custom/                  Lambda handler (alternative to HTTPS endpoint)
```

## Customization

The `--message` parameter controls the sender name shown in the notification. The `--count` parameter sets the number of unread messages.

```bash
./scripts/ping-me-multicast.sh "Kitchen" 3    # "You have 3 new messages from Kitchen"
./scripts/ping-me-multicast.sh "Reminder" 1   # "You have 1 new message from Reminder"
```

The actual notification text is generated by Amazon from the `AMAZON.MessageAlert.Activated` template — you can't send a custom message body with this schema.

## Limitations

- **Schema constraints**: The proactive events API uses predefined schemas. `AMAZON.MessageAlert.Activated` only supports sender name and message count — no free-form text.
- **Development stage**: Notifications are only delivered to the skill developer's own devices. To reach other users, the skill must go through certification.
- **Region-specific**: Use `--region eu` for European devices, `--region na` for North America, `--region fe` for Far East.
